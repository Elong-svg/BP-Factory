## 🔥 核心问题与解决方案

### 问题描述
用户反馈：各专家撰写商业计划书时图表和表格数量严重不足，流程监管专家和质量审核官检查不严格。

### 根本原因
1. **数量标准不明确** - 没有明确规定每个专家需要制作多少图表和表格
2. **检查机制不严格** - 流程监管专家和质量审核官检查流于形式
3. **专家职责不清** - 各专家不知道自己应该制作多少图表和表格
4. **缺乏自动化** - 依赖人工计数，容易疏漏

### 解决方案（v4.5.4）

#### 1. 明确最低数量标准
| 专家角色 | 图表最低数量 | 表格最低数量 | 关键图表 | 关键表格 |
|---------|------------|------------|---------|---------|
| 行业研究员 | ≥3 张 | ≥2 个 | 行业规模图、竞争格局图、趋势图 | 行业数据对比表、竞争对手对比表 |
| 财务分析师 | ≥3 张 | ≥3 个 | 营收趋势图、利润结构图、财务指标图 | 三大财务报表、财务指标表、同比分析表 |
| 战略分析师 | ≥2 张 | ≥2 个 | SWOT 矩阵图、战略规划图 | SWOT 分析表、战略规划表、投资评级表、风险提示表 |
| **全文总计** | **≥10 张** | **≥8 个** | - | - |

#### 2. 新增自动化计数脚本
**文件**：`scripts/count_charts_tables.py`

**功能**：
- 自动统计 Markdown 文档中的图片数量（`![]` 语法）
- 自动统计 Markdown 文档中的表格数量（`|` 语法）
- 分章节统计图表和表格数量
- 生成质量检查报告，标注达标/未达标章节
- 输出最终结论（通过/打回重做）

**使用方法**：
```bash
# 关口 2 检查（各专家章节）
python scripts/count_charts_tables.py chapters/industry.md
python scripts/count_charts_tables.py chapters/finance.md
python scripts/count_charts_tables.py chapters/strategy.md

# 关口 3 检查（整合后）
python scripts/count_charts_tables.py integrated/bp_merged.md

# 关口 4 检查（最终版）
python scripts/count_charts_tables.py final/bp_final.md
```

#### 3. 加强质量检查清单
**文件**：`quality-checklist-v4.5.4.md`

**关口 2（专家撰写完成）** - 18 项强制检查：
- ✅ 图表数量检查（流程监管专家重点监控）
- ✅ 表格数量检查（流程监管专家重点监控）
- ✅ 图表类型多样性检查
- ✅ 图表质量标准检查
- ✅ 表格质量标准检查

**关口 3（整合完成）** - 15 项强制检查：
- ✅ 图表数量达标检查（≥8 张）
- ✅ 表格数量达标检查（≥7 个）
- ✅ 图表计数验证（使用自动化脚本）
- ✅ 表格计数验证（使用自动化脚本）

**关口 4（最终检查）** - 20 项强制检查：
- ✅ 总图表≥10 张（**最低标准，必须达到**）
- ✅ 总表格≥8 个（**最低标准，必须达到**）
- ✅ 图表分布均匀（每章≥2-3 张）
- ✅ 表格分布均匀（每章≥2-3 个）
- ✅ 图表表格计数脚本已执行

#### 4. 明确各专家职责
**文件**：`专家职责_v4.5.4.md`

**行业研究员职责**：
- 图表≥3 张：行业规模图、竞争格局图、趋势分析图
- 表格≥2 个：行业数据对比表、竞争对手对比表
- 在撰写过程中同步制作，不得后期补做

**财务分析师职责**：
- 图表≥3 张：营收趋势图、利润结构图、财务指标图
- 表格≥3 个：三大财务报表、财务指标表、同比分析表
- 三大财务报表必须完整呈现

**战略分析师职责**：
- 图表≥2 张：SWOT 矩阵图、战略规划图
- 表格≥2 个：SWOT 分析表、战略规划表
- 撰写第七章"总结与投资建议"
- 第七章必须包含：投资评级表、风险提示表

#### 5. 严格惩罚机制
**图表表格数量不足特别处理**：
- 图表数量 < 10 张 → **直接打回重做，无需申辩**
- 表格数量 < 8 个 → **直接打回重做，无需申辩**
- 任何章节图表数量 = 0 → **该章节专家打回重做**
- 任何章节表格数量 = 0 → **该章节专家打回重做**

**流程监管专家职责**：
- 中期检查（撰写 50% 时）：图表≥2 张？表格≥1 个？
- 完成检查（撰写 100% 时）：全面验收
- 未达标立即叫停，要求补充

**质量审核官职责**：
- 关口 2、3、4 严格执行图表表格数量检查
- 对数量不足的章节行使一票否决权
- 记录每次检查的图表表格数量，形成质量档案

## 🎯 执行流程（v4.5.4）

### Step 1：数据收集与清洗
```
Claw 使用 finance-data-retrieval 获取数据
   ↓
调用 data-cleaner 清洗数据
   ↓
向用户报备数据情况
   ↓
流程监管专家检查（关口 1）
```

### Step 2：专家并行撰写（重点监控）
```
行业研究员撰写行业分析
   ├─ 每写 1 小节 → 检查是否有图表/表格支撑
   ├─ 完成 50% → 流程监管专家中期检查（图表≥2 张？表格≥1 个？）
   ├─ 完成 100% → 自我检查（图表≥3 张？表格≥2 个？）
   └─ 提交章节 + 数据来源清单

财务分析师撰写财务分析
   ├─ 每写 1 小节 → 检查是否有图表/表格支撑
   ├─ 完成 50% → 流程监管专家中期检查（图表≥2 张？表格≥2 个？）
   ├─ 完成 100% → 自我检查（图表≥3 张？表格≥3 个？）
   └─ 提交章节 + 数据来源清单

战略分析师撰写战略分析 + 第七章
   ├─ 每写 1 小节 → 检查是否有图表/表格支撑
   ├─ 完成 50% → 流程监管专家中期检查（图表≥1 张？表格≥1 个？）
   ├─ 完成 100% → 自我检查（图表≥2 张？表格≥2 个？第七章完整？）
   └─ 提交章节 + 数据来源清单

流程监管专家实时监控
   ├─ 中期检查（撰写 50% 时）
   ├─ 完成检查（撰写 100% 时）
   └─ 自动化计数验证（count_charts_tables.py）
```

### Step 3：整合完成检查
```
Claw 直接拼接各专家章节
   ↓
执行 count_charts_tables.py 脚本
   ↓
检查报告：图表≥8 张？表格≥7 个？
   ↓
是 → 进入关口 4
否 → 打回重做
```

### Step 4-5：最终检查与交付
```
执行 count_charts_tables.py 脚本（最终版）
   ↓
检查报告：图表≥10 张？表格≥8 个？
   ↓
是 → 质量审核官验收通过 → 调用 Word/PDF 技能生成最终文档
否 → 质量审核官一票否决 → 打回重做
```

## ✅ 质量标准

### 图表质量标准
每张图表必须包含：
- 标题（清晰说明图表内容）
- 坐标轴标签（X 轴、Y 轴含义和单位）
- 数据标签（具体数值或百分比）
- 图例（多数据系列时）
- 来源标注（数据来源说明）

图表类型多样性：
- 折线图（趋势分析）
- 柱状图（对比分析）
- 饼图/环形图（结构分析）
- 矩阵图（SWOT 分析等）
- 散点图（相关性分析）

### 表格质量标准
每个表格必须包含：
- 表头（清晰的列标题）
- 数据行（完整的数据）
- 单位标注（金额单位、百分比等）
- 来源说明（数据来源）

表格类型多样性：
- 对比表（多公司/多年度对比）
- 数据表（财务数据汇总）
- 分析表（财务指标分析）
- 总结表（关键结论汇总）
